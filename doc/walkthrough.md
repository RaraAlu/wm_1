# 全流程串讲：用大白话理解 LeWM / JEPA

---

## 先说清楚要解决什么问题

我们想训练一个机器人，给它看一张"目标图"，让它自己想办法把东西推到目标位置。

机器人没有人告诉它"应该往哪走"，它需要自己"想象"：  
> "如果我向左推，结果会是什么样？如果向右推呢？哪个更接近目标？"

**这个"在脑子里想象动作结果"的能力，就是世界模型（World Model）要学的东西。**

---

## 第一阶段：训练——让模型学会"想象"

### 用什么数据训练？

用的是人类专家操控机器人完成任务的录像，存在 `pusht_expert_train.h5` 里。  
每条录像里有：每一帧的图像、每一步执行了什么动作。

### 学什么？

不是让模型直接学"看图→输出动作"，而是让它学：  

> **"给我看前 3 帧图像 + 这 3 步执行了什么动作，你来猜第 4 帧图像的'特征'"**

注意：猜的不是像素，而是"特征"（嵌入向量）。就像你看一张猫的图，不用记住每个像素，只需要记住"这是一只橘色的、蜷缩着的猫"——这些特征信息才是重点。

### 具体怎么学？

**第一步：把图像压缩成特征向量**

用的是 ViT（Vision Transformer），专门设计用来理解图像的神经网络。  
把一张 224×224 的图像，压缩成一个 192 维的向量。

```
图像A（方块在左边） → [0.2, 0.8, -0.3, ...]  192个数字
图像B（方块在中间） → [0.5, 0.1,  0.9, ...]
图像C（方块在右边） → [0.3, 0.6,  0.2, ...]
图像D（方块到达目标）→ [0.7, 0.4, -0.1, ...]
```

动作也被压缩成同样维度的向量，这样图像和动作就在同一个"语言"里了。

**第二步：拼好"题目"和"答案"**

```
题目：图像A的特征 + 图像B的特征 + 图像C的特征
      动作"向右"  + 动作"向上"  + 动作"向左"

答案：图像B的特征、图像C的特征、图像D的特征
      （即：执行每步动作后，下一帧应该有什么特征）
```

**第三步：predictor 做预测**

predictor 是一个 Transformer，它读入题目，依次预测答案：

- 看到图像A + 动作"向右" → 猜图像B的特征
- 看到图像A、B + 动作"向上" → 猜图像C的特征
- 看到图像A、B、C + 动作"向左" → 猜图像D的特征

注意它是"因果"的——预测第2帧时，不能偷看第3帧（就像考试不能翻答案）。

**第四步：算差距，调整参数**

把预测的特征和真实的特征做对比（MSE 损失），差多少就往对的方向调整一点点。  
训练 100 轮，每轮看完所有数据，模型越来越准。

**还有一个额外的防崩手段（SIGReg）**

有个风险：模型可能"耍赖"——把所有图像都压缩成同一个向量，这样预测误差永远是零，但什么也没学到。  
SIGReg 正则项会检查：所有图像的特征是否足够"分散"，防止这种坍缩发生。

## 第二阶段：评估——用世界模型规划动作

### 两个角色分工明确

训练好的世界模型（JEPA）+ 规划算法（CEM）配合工作：

- **JEPA**：充当"预言机"。给它一个动作序列，它告诉你"如果执行这些动作，最终会有多接近目标"。
- **CEM（交叉熵方法）**：充当"搜索引擎"。它负责不断尝试不同的动作序列，找到代价最低的那个。

### CEM 规划的过程（一次完整规划）

**场景**：机器人现在看到当前画面，也知道目标画面，要规划接下来 5 步怎么走。

**第1轮（共30轮）**：

1. 随机生成 300 个"5步动作序列"，就像 300 个随机路线方案
2. 把这 300 个方案全都交给 JEPA："帮我预测每个方案执行后的样子"
3. JEPA 在脑子里"想象"300次，给每个方案打一个分（代价）
4. 选出分数最低（最接近目标）的 30 个方案
5. 分析这 30 个好方案的共同特征，在它们附近重新生成 300 个新方案

**第2~30轮**：重复上面的过程，每轮方案越来越聚焦，越来越好。

**结果**：30 轮后，找到一个相当不错的 5 步动作序列。

### JEPA 怎么"想象"一个动作序列的结果？

这是关键步骤，叫 **rollout（展开）**：

```
已知：当前 3 帧图像的特征 [e0, e1, e2]
要执行：5步动作 [a0, a1, a2, a3, a4]

JEPA 的推理过程（全在脑子里，不真的执行）：
  用 [e0,e1,e2] + a2 → 预测 e3（第4帧会是什么）
  用 [e1,e2,e3] + a3 → 预测 e4（第5帧会是什么）
  用 [e2,e3,e4] + a4 → 预测 e5（第6帧会是什么）

最后用 e5 和目标特征对比，差距越小越好。
```

注意：JEPA 始终在特征空间里推理，从不生成真实图像，速度非常快。

### 执行动作，再规划，循环

CEM 找到最优 5 步动作后：
- 执行这 5 步（每步实际重复 5 次，共 25 帧）
- 执行完毕，观察新的当前画面
- 触发下一轮 CEM 规划（基于新的当前状态）
- 循环，直到完成任务或超出步数限制（50步）

## 用一个比喻把整体串起来

想象你在玩一个"推箱子"游戏，但是蒙着眼睛，只能靠一个"顾问"告诉你结果好不好。

**训练阶段** = 培养这个顾问  
让顾问看了大量高手的录像，学会"如果在这种局面下往右推，接下来大概会变成什么局面"。

**评估阶段** = 正式上场  
你（CEM）想了 300 种走法，全部问顾问（JEPA）："这种走法最后结果怎么样？"  
顾问每次都在脑子里模拟一遍，给你一个分数。  
你不断迭代，越问越精准，最后找到最优方案，然后走这几步，再问，再走……  
直到把箱子推到目标位置。

**关键优势**：顾问只需要在特征空间里"想象"，不需要真的模拟每一帧像素，速度快、效率高。

## 整体时间线

```
专家录像数据
    │
    │  （离线，一次性）
    ▼
训练 100 个 epoch
    ├─ 模型学会：特征压缩（ViT + projector）
    └─ 模型学会：特征预测（predictor + pred_proj）
    │
    ▼
保存模型权重（checkpoint）
    │
    │  （在线，实时）
    ▼
机器人进入环境
    │
    ▼
┌─────────────────────────────────────────────────┐
│  看到当前画面 + 目标画面                         │
│      │                                          │
│      ▼                                          │
│  CEM 规划（30轮 × 300个候选 = 9000次JEPA推理）  │
│      │                                          │
│      ▼                                          │
│  执行最优 5 步动作                               │
│      │                                          │
│      └──────────────── 循环 ◀─────────────────┘
│                                                 │
▼
任务完成 / 超时结束，记录得分
```
